优点

  • 静态文件安全、备份方便

  • WordPress（运行在php上的）的缺点：

    • 备份：不断备份数据库。

    • 插件：实现功能需要不同插件，不断升级。

    • 升级：不断升级来确定安全性。

  • Markdown文件管理及方便

    • 纯文本

    • 格式统一

  • Windows下载方便，单文件的

    • 其他的同类jekyll、HEXO

已知缺点

  • 要确认申请的主机是否支持中文语言（URL）

安装

  • 到 Hugo Releases 下载对应的操作系统版本的Hugo二进制文件

  • 首先安装好依赖的工具：

    • Git
    • Mercurial
    • Go 1.3+ (Go 1.4+ on Windows)

    设置好 GOPATH 环境变量，获取源码并编译：

    $ export GOPATH=$HOME/go
$ go get -v github.com/spf13/hugo

    源码会下载到 $GOPATH/src 目录，二进制在 $GOPATH/bin/

    如果需要更新所有Hugo的依赖库，增加 -u 参数：

    $ go get -u -v github.com/spf13/hugo

生成站点

使用Hugo快速生成站点，比如希望生成到 /path/to/site 路径：

$ hugo new site /path/to/site

在 /path/to/site 目录里生成了初始站点，站点目录结构：

  ▸ archetypes/
  ▸ content/
  ▸ layouts/
  ▸ static/
    config.toml

接下来需要完成以下几步：

  1. Change the current directory to G:\HuGo\hugo\HZW.

    （切换路径到站点目录下）

  2. Create or install a theme:

    • Create a new theme with the command “hugo new theme ”
    • Install a theme from https://themes.gohugo.io/

    （安装或者创建一个主题，可以去找现成的 https://www.gohugo.org/theme/）

  3. Edit hugo.toml, setting the “theme” property to the theme name.

    (配置toml，可以直接复制现有主题下的.toml文件)

    注意toml文件名要和theme的名字相同

  4. Create new content with the command “hugo new content <FILENAME>.”.

    （支持多个格式，不只是 md）

  5. Start the embedded web server with the command “hugo server –buildDrafts”.

安装皮肤

到 皮肤列表 挑选一个心仪的皮肤，比如你觉得 Hyde 皮肤不错，找到相关的 GitHub 地址，创建目录 themes，在 themes 目录里把皮肤 git clone 下来：

# 创建 themes 目录
$ cd themes
$ git clone https://github.com/spf13/hyde.git

或者直接复制粘贴

运行Hugo

在你的站点根目录执行 Hugo 命令进行调试：

$ hugo server --theme=主题名 --buildDrafts

浏览器里打开： http://localhost:1313

或者直接hugo server 主题在config.toml里直接设置好，就会自动启动默认的主题

创建文章

创建一个 about 页面：

$ hugo new about.md

about.md 自动生成到了 content/about.md ，打开 about.md 看下：

+++
date = "2015-10-25T08:36:54-07:00"   #日期
draft = true        #是否是草稿
title = "about"     #文章标题

+++

正文内容

内容是 Markdown 格式的，+++ 之间的内容是 TOML 格式的，根据你的喜好，你可以换成 YAML 格式（使用 --- 和...标记）或者 JSON 格式。

默认情况下，都是草稿，即draft=true,可以直接更改或者把它删掉，或者在添加-D/--buildDrafts参数

部署

假设需要部署在 GitHub Pages 上，首先在GitHub上创建一个Repository，命名为：用户名.github.io （用户名替换为你的github用户名，必须小写）。

在站点根目录执行 Hugo 命令生成最终页面：

$ hugo --theme=主题名 --baseUrl="http://<yourname>.github.io/"

（注意，以上命令并不会生成草稿页面，如果未生成任何文章，请去掉文章头部的 draft=true 再重新生成。）

如果一切顺利，所有静态页面都会生成到 public 目录，将pubilc目录里所有文件 push 到刚创建的Repository的 master 分支。

$ cd public
$ git init
$ git remote add origin https://github.com/coderzh/coderzh.github.io.git
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master

浏览器里访问：http://coderzh.github.io/

但是同时 Hugo 还提供了一些选项。

  1. 这些 flags 是非常大小写敏感的！（Hugo v0.15版本）

草图（Sketches）（暂译）

sketches是 heyo 侧边栏中的漂亮可视化效果。默认的都是使用 p5.js 构建的——一个用于创意编码的 javascript 库。

目前有以下几种效果：

  • Graph图（类似博客园里的背景，点和线构成的图）
  • Digital Rain数字雨
  • Circles圈

建立

全局配置

可以在config.toml中的 params.sketch 配置

  • enable：控制草图是否显示在每一页上
  • displayOptions：控制草图设置是否在悬停时显示
  • use：选择将显示的草图

每页基础

即使将enable=false，仍然可以在特定页面上启用草图。只需在页面的front-matter（前言）中设置。

实现自己的草图

可以导入自己的JS代码（之前讲了），只需要在JS文件中：

  1. 在类（名为yourclass）中描述 sketch
  2. 确保它实现了setup 和 draw（也可以实现mouseWheel， mouseClicked 和 getSettings）
  3. 将类添加到对象SKETCHES 中的 SKETCHES['<yourName>'] = <yourclass>
  4. 设置sketch.use为yourName

帖子缩略图(Thumbnail)

给帖子添加缩略图，只需将变量 thumbnail 添加到front-matter。对于thumbnail，可以定义：

  • src：图片来源
  • alt：替换值，（当图片失效时显示），默认为thumbnail
  • object_postion：图像位置（默认为50%，50%）
  • height：缩略图的高度（默认为250px） 默认值可以在CSS变量 --thumbnail-size 中更改，这个变量就在customCSS所指向的文件中

徽章(badges)

徽章是使用 badgen.net 生成的。它们可以使用shortcode构建，并且还有一种方法可以通过 front-matter 将多个徽章添加到post summary中。

可以post summary中显示徽章， 通过在front-matter中设置变量 badges ，例如：

badges:
	github:
		subject: GitHub
		status: Check it on GitHub
    icon: github
    url: https://github.com/LucasVadilho/heyo-hugo-theme
    color: grey
    label: ""
  template:
  	flat: false
  	subject: subject
  	status: status
  	label: label
  	color: pink
  	label_color: 000
  	icon: awesome
  	url: https://badgen.net/

参数

参数 描述 值 默认
status 右边的文本或者generator的设置 Any string status
subject 左边的文字或者generator的设置 Any string subject
generator 徽章生成器 Checkout badgen’s documentation static
flat 徽章样式 `true false`
scale 徽章比例 Any number 1
label 左侧的文字 Any string None
url 点击跳转到特定URL Any URL None
icon 左侧的图标 Bultin 图标 or any SVG URL Badgen 的默认值
color 状态颜色 Bultin 颜色 or 十六进制代码 Badgen 的默认值
label_color 标签颜色 Bultin 颜色 or 十六进制代码 Badgen 的默认值

subject会被label给覆盖

label可以为空，这样只会显示图标
